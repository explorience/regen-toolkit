# Teaching Wallet Security

*How to teach wallet security effectively to community members.*

---

## Overview

Wallet security is the most critical topic in blockchain education. A single mistake — sharing a seed phrase, clicking a phishing link — can result in permanent, irreversible loss. Teaching security well is about building habits, not just transferring knowledge.

## Core Security Concepts to Teach

### 1. Seed Phrases Are Sacred
Your seed phrase is the master key to your wallet. Never share it. Never type it into a website. Never store it digitally. Write it on paper, store it safely.

**Teaching approach**: Use the "house key" analogy. Your seed phrase is like the master key to your house. You wouldn't post it on social media.

### 2. Verify Everything
Before signing a transaction, verify:
- The recipient address (check first and last characters)
- The amount
- The network (Ethereum? Base? Polygon?)

**Teaching approach**: Practice with test transactions. Show what a phishing attempt looks like.

### 3. Start Small
First transactions should be tiny amounts. Learn on $5, not $500.

**Teaching approach**: Practice with testnet tokens first, then small real amounts.

### 4. Hardware Wallets for Significant Amounts
If you're holding more than you'd carry in your physical wallet, use a hardware wallet (Ledger, Trezor).

**Teaching approach**: Demonstrate hardware wallet setup in a workshop. Let people hold the device.

## Common Security Threats

### Phishing
Fake websites that look like real ones. "Connect your wallet" on a lookalike site.

**Prevention**: Bookmark legitimate sites. Never click links in DMs.

### Social Engineering
"I'm from support, I need your seed phrase to fix your account."

**Prevention**: No legitimate service ever asks for your seed phrase. Ever.

### Clipboard Hijacking
Malware that changes copied addresses to the attacker's address.

**Prevention**: Always verify the address after pasting, not just before copying.

## Teaching Methods

1. **Live demonstrations**: Show real phishing attempts (safely)
2. **Practice exercises**: Set up wallets, send test transactions
3. **Scenario-based learning**: "What would you do if..."
4. **Peer teaching**: Members teach each other
5. **Cheat sheets**: Laminated security reminders

## Moving Forward

Security is not a one-time lesson. Build security awareness into every session. Review practices regularly. Create a culture where asking "is this safe?" is encouraged.
