Rather than writing articles linearly (1.1 â†’ 1.2 â†’ 1.3...) or haphazardly, identify **critical paths** for real use cases. Each path = minimum viable toolkit for that audience.

## Implementation Ideas

### Option A: Explicit Paths in UI
- Homepage shows "Choose Your Path"
- Each path is a guided journey
- Progress tracking per path
- "You've completed 8/20 articles in the Tree Planter path"

### Option B: Tagged Articles
- Articles tagged with relevant paths
- Filter by path in search/browse
- "This article is part of: Global South Youth, Silvi"

### Option C: Internal Prioritization Only
- Use paths to decide writing order
- Don't surface to users
- Linear structure remains
